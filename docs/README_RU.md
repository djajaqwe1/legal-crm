# CONGLOMERATE AI Legal CRM MVP — инструкция

## 1) Что это за сервис

Текущая версия — это **MVP интерфейс Legal CRM** для юридической команды:
- дашборд и реестр дел;
- навигация по ключевым разделам;
- базовая структура хранения документов: `Клиент -> Объект -> Дело`;
- foundation для дальнейшей автоматизации процессов (агенты, интеграции, конструктор документов).

## 2) Что уже реализовано

- Стек: `Next.js 16 (App Router) + Tailwind CSS + shadcn/ui`.
- UI-структура:
  - `Дашборд` (`/`)
  - `Клиенты` (`/clients`)
  - `Реестр дел` (`/cases`)
  - `Конструктор документов` (`/documents-builder`)
- Главная страница CRM с таблицей дел:
  - статус;
  - клиент;
  - дедлайн.
- LegalTech стиль интерфейса: сдержанная цветовая схема, строгая типографика, светлая/темная палитра.
- Исправлены:
  - гидратационный warning от браузерной инъекции (через `suppressHydrationWarning`);
  - статичные кнопки sidebar заменены на реальную маршрутизацию.

## 3) Как запустить локально

```bash
cd legal-crm
npm install
npm run prisma:generate
npm run dev
```

После запуска открыть: [http://localhost:3000](http://localhost:3000)

## 4) Как пользоваться (сценарий демо)

1. Открой `Дашборд`.
   - Покажи таблицу активных дел: статус, клиент, дедлайн.
2. Перейди в `Клиенты`.
   - Покажи список клиентов и ответственных.
3. Перейди в `Реестр дел`.
   - Покажи централизованный реестр для операционной работы.
   - Измени статус дела прямо в таблице (например, на `Завершено`).
4. Перейди в `Конструктор документов`.
   - Покажи точку роста MVP под генерацию исков и шаблонов.
5. Подчеркни архитектуру хранения:
   - `Клиент -> Объект -> Дело` (готово в UI и как бизнес-стандарт структуры).

## 5) Что показывать заказчику прямо сейчас

- Готовый рабочий интерфейс MVP (не мокап).
- Навигацию между разделами без перезагрузки.
- Базовую модель организации юридической работы.
- Подготовленность к следующему этапу автоматизации (интеграции и AI-агенты).

## 5.1) Базовый рабочий процесс юриста

1. В `Клиенты` создается карточка клиента.
2. В `Реестр дел` добавляется дело и назначается статус.
3. По ходу работы статус обновляется: `Новый -> В работе -> Суд -> Завершено` (или `Пауза`).
4. Дашборд и реестр отражают актуальное состояние по делам и срокам.

## 6) Ограничения текущей версии

Сейчас это MVP с подключенной доменной моделью Prisma. Пока нет:
- авторизации и ролей;
- живых интеграций с OneDrive/почтой/гос.базами;
- автоматической генерации документов;
- AI-оркестрации агентов.

Подробная сверка с ТЗ и roadmap: `docs/TZ_MVP_ANALYSIS_RU.md`.

## 7) PostgreSQL и сиды (этап A)

1. Подними PostgreSQL (локально или в облаке).
2. В `.env` задай `DATABASE_URL`.
3. Прогони миграцию и сиды:

```bash
npm run prisma:migrate
npm run prisma:seed
```

После этого страницы `Клиенты` и `Реестр дел` будут работать с реальной БД, а формы начнут создавать записи в PostgreSQL.

## 8) Gemini AI-помощник по делу

1. Добавь ключ Gemini в `.env`:

```env
GEMINI_API_KEY="your_api_key_here"
```

2. Перезапусти сервер:

```bash
npm run dev
```

3. В `Реестр дел` нажми `Открыть чат` у нужного дела.
4. Задай вопрос по делу — ассистент отвечает по контексту конкретной карточки.
